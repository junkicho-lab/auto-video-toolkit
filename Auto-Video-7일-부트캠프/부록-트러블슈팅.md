# 🔧 부록 — 트러블슈팅

> [!tip] 만능 해결책
> 막히면 **에러 메시지를 그대로 복사해서 Claude Code에게 붙여넣고 물어보세요.** "이 에러 왜 나? 어떻게 고쳐?" 대부분 해결됩니다.

---

## 설치 / 환경 (Day 2)

> [!failure] `command not found: brew`
> Homebrew 미설치. [[Day-2-환경설정]] Step 1의 설치 스크립트 실행. 설치 후 터미널을 껐다 켜세요.

> [!failure] `pip install` 중 에러
> 가상환경이 켜져 있는지 확인 (`(.venv)` 표시). 안 켜졌으면 `source .venv/bin/activate`. 그래도 안 되면 `python3 -m pip install --upgrade pip` 후 재시도.

> [!failure] `import` 에러 (No module named ...)
> 패키지 미설치. `pip install -e ".[dev]"`를 가상환경 안에서 다시 실행.

---

## 이미지 생성 (ima2)

> [!failure] `ima2 ping` 실패 / 이미지가 안 나옴
> 1. **다른 터미널에서 `ima2 serve`가 실행 중인지** 확인 (가장 흔한 원인)
> 2. `npx @openai/codex login` 을 했는지 확인
> 3. 그래도 안 되면 serve 터미널을 끄고 다시 `ima2 serve`

> [!failure] `ima2: command not found`
> `npm install -g ima2-gen` 재실행. 또는 `cd vendor/ima2-gen && npm install && npm link`.

> [!failure] 이미지마다 인물(인종/얼굴)이 달라짐
> **프롬프트에 인종·피부색을 명시**하고 `--anchor`로 앵커 이미지를 전달하세요. → [[Day-6-Phase2-비주얼과음성]]

---

## 영상 생성

> [!failure] "필요한 키 미설정" / provider 에러
> 선택한 provider의 키가 `config/api-keys.env`에 있는지 확인. provider는 `config/models.yaml`의 `video_generation` 값과 `--provider` 옵션으로 결정됩니다.

> [!failure] 영상 클립이 너무 비싸다
> 첫 영상은 모든 씬을 `image`로 만들고 합성 시 Ken Burns 효과로 움직임을 주세요. 영상 클립은 나중에.

---

## 나레이션 (TTS)

> [!failure] 나레이션이 0초로 생성됨
> 1. `ELEVENLABS_API_KEY`와 크레딧 잔량 확인
> 2. (Voicebox 로컬 사용 시) 모델이 언로드되면 0초가 나옴 → `/history/{id}`로 폴링 확인 (`/generate/{id}/status` 아님)
> 3. `ffprobe`로 길이 확인: `ffprobe -i ...wav 2>&1 | grep Duration`

---

## 합성 / CapCut (Day 7)

> [!failure] `ffprobe 없음`
> `brew install ffmpeg` 실행.

> [!failure] CapCut에서 미디어가 안 보임 / 깨짐
> macOS CapCut은 샌드박스라 외부 파일을 못 읽습니다. **반드시 `capcut-project-gen.py`를 사용**하세요 (Resources/ 복사 + draft_info.json 자동 생성). 수동 생성 금지.

> [!failure] 자막이 깨지거나 안 나옴
> SRT는 **플레인 텍스트만 + 자막 1개당 1줄** 규칙을 지켜야 합니다. 메타데이터/2줄 자막 금지. → [[Day-6-Phase2-비주얼과음성]]

> [!failure] Windows에서 합성이 안 됨
> CapCut 자동 합성은 macOS 기준입니다. Windows는 영상·나레이션·자막까지 만든 뒤 수동 합성하세요.

## Phase / 에이전트

> [!failure] Phase 2가 시작 안 됨
> `brand/` YAML 4개가 모두 채워져야 합니다. 비었으면 [[Day-4-Phase1-브랜드세팅]] 먼저.

> [!failure] 에이전트 결과가 마음에 안 듦
> 체크포인트에서 **수정 요청**으로 구체적 피드백을 주고 재실행. 3회 넘으면 방향 자체를 재검토하세요.

## 비용 관리 (중요)

> [!warning] 과금을 줄이는 습관
> - 연습은 **숏폼 59초**부터
> - 이미지는 `--start 1 --end 2`로 **소수 씬 먼저 테스트**
> - 첫 영상은 영상 클립 대신 **이미지 + Ken Burns**
> - `--quality medium`으로 시험 → 만족 시 `high`
