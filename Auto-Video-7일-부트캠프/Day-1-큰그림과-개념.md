# Day 1 — 큰 그림과 핵심 개념

> [!success] 오늘의 목표
> 코드를 한 줄도 건드리지 않고, **"이 프로젝트가 무엇을, 어떻게 만드는지"** 전체 그림을 머릿속에 그립니다. 오늘은 **읽고 이해**하는 날입니다.

선행: [[00-시작하기]] · ⏱️ 예상 90분

## 1. 한 문장 정의

> **Auto-Video = "AI 일꾼(에이전트) 여러 명이 역할을 나눠, 유튜브 영상 한 편을 자동으로 만들어주는 공장(파이프라인)."**

사람이 영상 한 편을 만들 때 하는 일을 떠올려 보세요:

1. 주제를 정하고 → 2. 자료 조사하고 → 3. 대본 쓰고 → 4. 장면을 나누고 → 5. 그림/영상 만들고 → 6. 목소리 녹음하고 → 7. 자막 달고 → 8. 편집해서 합칩니다.

이 8단계를 **각각 다른 AI 일꾼**에게 맡기고, **팀장(Director)** 이 순서대로 지휘하는 것이 바로 이 프로젝트입니다.

---

## 2. 가장 중요한 개념: "에이전트 vs 스크립트"

이 두 가지만 구분하면 절반은 이해한 겁니다.

> [!note] 비유: 요리사와 주방기구
> - **에이전트(`agents/*.md`)** = 요리사 🧑‍🍳 — "무엇을, 어떻게 만들지" **판단**합니다. 사실은 그냥 **잘 쓰여진 지시서(프롬프트 텍스트)**입니다.
> - **스크립트(`scripts/*.py`)** = 주방기구 🔪 — 실제로 **이미지를 생성하고, 영상을 뽑고, 합치는 손발**입니다. Python 코드입니다.
>
> 요리사(에이전트)가 "이 장면 그려줘"라고 판단하면, 주방기구(스크립트)가 실제로 그림을 만듭니다.

| 구분 | 정체 | 역할 | 위치 |
|---|---|---|---|
| 에이전트 | 마크다운 텍스트(프롬프트) | 판단·창작·지시 | `agents/` |
| 스크립트 | Python 코드 | 실제 실행(API 호출) | `scripts/` |

---

## 3. 2-Phase 파이프라인 (핵심 구조)

이 공장은 **2개의 단계(Phase)**로 나뉩니다.

### 🏗️ Phase 1 — 브랜드 세팅 (채널당 딱 1번)

> 채널의 "정체성"을 만드는 단계. 한 번만 하면 됩니다.

```
brand-architect → narrative-designer → topic-strategist → benchmark-analyst
   (캐릭터)          (서사/세계관)        (주제 전략)         (벤치마크 분석)
```

→ 결과물: `brand/` 폴더에 **YAML 4개** (`character`, `narrative`, `topics`, `benchmark`)

### 🎥 Phase 2 — 콘텐츠 제작 (영상 1편당 1번)

> 실제 영상을 뽑는 단계. 영상 만들 때마다 반복합니다.

```
researcher → scriptwriter → scene-director → [image-generator + video-generator] → voice-producer → compositor
 (리서치)      (대본)          (씬 분할)          (이미지/영상 — 병렬 실행)            (나레이션)      (합성)
```

→ 결과물: `projects/{날짜-주제}/` 폴더에 대본·이미지·영상·나레이션·자막·최종 mp4

> [!tip] 외우는 법
> **Phase 1 = 채널을 "세팅"** (1회) / **Phase 2 = 영상을 "찍기"** (매번). 식당으로 치면 Phase 1은 *간판·메뉴 정하기*, Phase 2는 *손님 올 때마다 요리하기*.

---

## 4. 또 하나의 핵심: "Video-First" 사고

보통은 대본을 먼저 쓰지만, 이 프로젝트는 **거꾸로** 합니다.

> [!important] 왜 거꾸로?
> AI 영상 도구마다 만들 수 있는 클립 길이가 정해져 있기 때문입니다. (예: Kling=10초, Veo3=8초) 그래서 **러닝타임을 먼저 정하고 → 씬 수를 역산 → 대본을 거기에 맞춤**.

```
1. 러닝타임 정하기        (예: 3분 30초 = 210초)
2. 영상 도구 선택         (예: Veo3 → 클립당 8초)
3. 씬 수 = 210 ÷ 8       ≈ 26씬
4. 총 단어 = 210 ÷ 60 × 150  = 525단어   (영어 150 WPM 기준)
5. 씬당 단어 = 525 ÷ 26   ≈ 20단어
6. 앵커→이미지→영상→대본→TTS→합성
```

이 계산은 Day 5에서 직접 해봅니다. 지금은 "**길이부터 정하고 역산한다**"만 기억하세요.

## 5. 결과물 폴더 구조 미리보기

영상 1편을 만들면 이런 폴더가 생깁니다:

```
projects/{slug}/
├── scenes/
│   ├── images/      # scene-01.png ~ scene-N.png
│   └── videos/      # scene-01.mp4 ~ scene-N.mp4
├── anchors/         # 캐릭터 기준 이미지(일관성용)
├── narration/       # scene-01.wav ~ scene-N.wav
├── script.md        # 영문 대본
├── script-ko.md     # 한글 검수용 대본 (영어 콘텐츠 시)
├── image-prompts.md
├── video-prompts.md
├── capcut-config.json
└── capcut-export/subtitles.srt
```
